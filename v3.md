# Odoo Cafe Prototype - V3 Documentation

This document outlines the architecture, page logic, and new features implemented in the V3 version of the Odoo Cafe application. 

## 1. Overview of V3 Updates
The V3 update focuses on connecting the frontend prototype to the backend SQLite database, improving real-time synchronization, and polishing the user and employee experience. Key updates include:
*   **Database Integration:** Orders and payments are now persistently stored in the backend SQLite database instead of just local memory.
*   **Real-Time Analytics:** The Executive Analytics Dashboard now calculates live metrics from actual orders instead of displaying mock data.
*   **Smart Table Management:** Tables are automatically freed when a customer logs out, and admins have a quick "Reset Tables" button.
*   **Customer Notifications:** Customers receive real-time, auto-dismissing toast notifications on their dashboard when their order is marked "Delivered".

---

## 2. Core State & Data Flow

### Backend Database (SQLite & FastAPI)
*   **`/api/orders/`**: Handles the creation of an order. Saves the total, tax, items, and table information to the DB, and instantly broadcasts a `NEW_ORDER` websocket event to the Kitchen Display System (KDS).
*   **`/api/orders/{id}/pay`**: Updates the transaction status in the database to "Paid".

### Frontend Stores (Zustand)
*   `useOrderSyncStore`: Uses `BroadcastChannel` to keep order status in sync across different browser tabs (Customer Dashboard vs Employee Dashboard) in real-time.
*   `useTableStore`: Tracks which tables are available vs occupied. Persists to local storage so table selection survives page reloads.
*   `useAuthStore`: Manages user login sessions, roles, and handles freeing up tables automatically upon logout.

## 3. Page Logics & Key Buttons

### 1. Landing Page (`/`)
The entry point for customers to select a table or log in.
*   **"Dine In - Choose Table" Button**: Reveals the interactive table map.
*   **Table Selection Cards**: Clicking an available table reserves it for the current user and navigates them to the POS.
*   **"Logout" Button (Header)**: Clears the user's session and **automatically frees up any table** they were currently occupying.
*   **"Dashboard" Button (Header)**: Routes the user to either the Customer Portal or Admin Layout based on their role.

### 2. Point of Sale & Checkout (`/pos`, `/checkout`)
The main ordering interface for customers.
*   **"Add to Bag" Buttons**: Adds products to `cartStore`.
*   **"Proceed To Checkout"**: Transitions the user to the cart review page.
*   **"Pay Now" (Payment Modal)**: 
    *   Fires `ordersService.createOrder()` to save the transaction in the backend DB.
    *   Triggers `ordersService.payOrder()` upon successful mock payment.
    *   Dispatches the order to `orderSyncStore` for real-time tracking across dashboards.

### 3. Customer Dashboard (`/admin/customer`)
The portal where customers can track their history and active orders.
*   **Order History List**: Displays past and current orders.
*   **Real-Time Delivery Toast**: A `useEffect` hook constantly monitors the `orders` array. If an order transitions to "Delivered", a green floating notification appears at the top of the screen ("Order #<id> has been delivered! Enjoy!").
*   **"Home" Button**: Quickly returns the customer to the main catalog.

### 4. Employee Dashboard (`/admin/employee`)
The control center for staff to manage incoming front-of-house operations.
*   **"Confirm & Dispatch" Button**: Moves an order from "Pending" (just paid) to "Sent", routing it to the kitchen queue.
*   **"Mark Delivered" Button**: Once the kitchen completes an order, the employee clicks this to notify the customer.
*   **"Reset Tables" Button**: A utility button in the header that forcefully clears the `occupied` status of all tables, making them available for new walk-ins.
*   **"Publish Promo"**: Creates active discount codes that sync to the `promoStore`.

### 5. Executive Analytics Dashboard (`/admin/dashboard`)
The manager's view for tracking cafe performance.
*   **Live Metrics**: Calculates Total Revenue, Total Orders, and Avg Order Value dynamically from the synchronized order data.
*   **Sales Trend Chart**: Generates a dynamic SVG bar chart by distributing real revenue across time intervals.
*   **Top Orders & Top Products Tables**: Aggregates the items from live orders to show exactly what is selling best during the current session.
*   **"Export CSV/PDF"**: Triggers local file downloads of the sales reports.

### 6. Kitchen Display System (KDS)
The back-of-house screen for chefs.
*   **Websocket Connection**: Listens to `ws/kds`. When the backend creates an order, it instantly appears here.
*   **"Bump" / "Complete" Button**: Moves the ticket out of the active kitchen queue and updates the order status to "Completed", notifying the Employee Dashboard that the food is ready to run.

## 4. The Complete Order Lifecycle
1. **Selection:** Customer picks a table on the Landing Page.
2. **Checkout:** Customer builds a cart and pays. The backend database saves the order, assigning an Integer ID.
3. **Kitchen Sync:** The backend fires a websocket event; the KDS receives the ticket.
4. **Employee Verification:** The employee sees the order on their dashboard and dispatches it.
5. **Preparation:** The kitchen prepares the food and clicks "Complete" on the KDS.
6. **Delivery:** The employee physically runs the food to the table and clicks "Mark Delivered" on their dashboard.
7. **Notification:** The customer's dashboard instantly pops up a toast notification confirming delivery.
8. **Departure:** The customer logs out, instantly freeing up their table for the next guest.
